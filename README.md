# Browser Palette Reducer

GitHub Pagesで公開できる, ブラウザ内完結の色数削減アプリです. 

日本人初心者でも直感的に使えるよう, **「①動画を選ぶ → ②色を減らす → ③確認する → ④保存する」のかんたん4ステップUI**に刷新しました（フォントは Zen Kaku Gothic New）. **複数の動画をまとめて一括処理**できます. UIの設計意図は `docs/design-brief.md`, 画面要素の解説は `docs/app-guide.md` を参照してください. 

## できること

- ローカル動画を**複数まとめて**ブラウザで読み込む（ドラッグ＆ドロップ対応）
- すべての動画に同じ設定を適用し, 順番に一括分析する
- プリセット（おすすめ / 高精細 / 軽量）で数値を触らず開始できる. 細かい調整は「詳細設定」に格納
- 最初のフレームと最後のフレームから主要色を抽出する
- RGB空間内で自動的に代表色数を選ぶ
- 代表色パレットを表示する（クリックで HEX をコピー）
- 元動画, 色数削減後, はみ出し色マップ（新色マスク）を同期プレビューする
- 色の3次元プロット（RGB空間）を回転表示する
- しきい値をスライダーで調整し, はみ出し色（マゼンタ）の量をその場で確認する
- 保存対象の動画を選んで, 処理後動画をWebMとして一括書き出し・ダウンロードする
- 分析・書き出しを中断する
- Chromeが直接読めない動画は, ブラウザ内のffmpeg.wasmでH.264 MP4へ変換してから処理する

## 制約

- サーバ側処理はありません. 
- 動画はアップロードされず, ブラウザ内で処理されます. 
- Chrome / Edge を優先対象にしています. 
- MP4での書き出しではなく, WebMでの書き出しです. 
- 大きい動画では処理時間が長くなります. 
- 非対応動画の自動変換では, 初回に約31MBのWASMファイルを読み込みます. 
- ffmpeg.wasmでもデコードできない特殊な動画は処理できません. 
- 既存のPython版と完全に同じ結果を保証するものではありません. 

Python版では候補色5,000色で解析していました. ブラウザ版で同じ規模の階層クラスタリングを行うと重くなるため, 初期値は候補色800色にしています. 精度を上げたい場合は `Max colors` を増やしてください. ただし, 増やすほど処理時間とメモリ使用量が増えます. 

## 推奨初期設定

- Analysis short side: `540`
- Preview short side: `360`
- Max colors: `800`
- Min rep distance: `12`
- Threshold mode: `Auto`

精度を上げたい場合は `Max colors` と `Analysis short side` を上げます. ただし処理は重くなります. 

## 確認方法

解析後に `Preview` を押すと, キャンバス上で元動画, 処理後動画, 新色マスクが同期して再生されます. 

`Export WebM` が完了すると, `Exported WebM` パネルに書き出し済み動画が表示されます.
