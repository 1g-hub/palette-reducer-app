# Browser Palette Reducer — アプリ解説

このドキュメントは `palette-reducer-app` ディレクトリの中身と、GUI（画面に出る表示要素）を解説するものです。コードを読まなくても、このアプリが「何をするもので、画面のどこに何が出るのか」が分かることを目的にしています。

> ⚠️ **補足（2026-06-17 更新）**: GUI は「かんたん4ステップ＋複数動画の一括処理」のデザインへ刷新済みです（`docs/design-brief.md` の方針を実装）。新しい画面構成は同ブリーフの「5. 画面フロー」を参照してください。本書の「4. GUI 表示要素の解説」は**刷新前のレイアウト**を記録したものですが、各表示要素（パレット／3映像／Metrics／K候補表／各設定）の**意味の説明は新UIでもそのまま有効**です（新UIでは設定はプリセット＋詳細設定に、Metrics/K候補表は「詳細データ」に集約されています）。

---

## 1. このアプリは何か

**ブラウザの中だけで完結する「動画の色数削減（パレット化）」ツール**です。

- 動画を選ぶと、その動画で使われている色を分析して、**少数の代表色（パレット）に置き換えた動画**を作ります。
- どの色にも当てはまらない「想定外の色（新しい色）」は**マゼンタ（鮮やかなピンク）**で塗って目立たせます。
- 処理はすべて**あなたのブラウザの中**で行われます。動画はサーバーに送られません（アップロードなし）。
- サーバー不要なので、GitHub Pages のような静的ホスティングだけで動きます。

### 典型的な使い道
- 動画の色を「決められた数色」に統一したい（デザイン・印刷・刺繍・LED表示・ドット絵などの前処理）。
- 「この動画には、想定したパレット以外の色がどれくらい紛れているか」を可視化したい。

## 2. ファイル構成

| ファイル / フォルダ | 役割 |
| --- | --- |
| `index.html` | 画面の骨組み（HTML）。GUI 要素はすべてここで定義。 |

| `.nojekyll` | GitHub Pages で Jekyll 処理を無効化するための空ファイル。 |

## 3. 処理の流れ（裏側で起きていること）

1. **動画読込** — ローカル動画を選ぶ。
2. **フレーム抽出** — 動画の「最初のフレーム」と「最後のフレーム」を取り出す。
3. **色のカウント** — その2枚から、使われている色を数え上げる（ヒストグラム）。
4. **候補色の絞り込み** — 出現数の多い順に、上位 `Max colors`（既定800色）だけを候補にする。
5. **クラスタリング** — RGB 空間で似た色をまとめる階層クラスタリング（complete-linkage）を `worker.js` が実行。
6. **代表色数 K の自動決定** — まとまり具合（relative gap）を見て、適切な色数を自動で選ぶ。`Min rep distance` を満たす範囲に限定。
7. **しきい値の決定** — 各画素が「いちばん近い代表色」からどれだけ離れているかを測る基準値を決める（Auto はデータから自動算出、Fixed は手入力）。
8. **プレビュー** — 元動画／削減後／新色マスクの3画面を同期再生。
9. **書き出し** — 削減後の動画を WebM で出力。
10. **（必要時）変換** — ブラウザが読めない動画は ffmpeg.wasm で先に変換してから処理。

> 補足: 元になった Python 版は候補色5,000色で解析していましたが、ブラウザで同じ規模をやると重いため、初期値は800色に下げています（`README.md` 参照）。

## 4. GUI 表示要素の解説（ここが本題）

画面は大きく **左の「操作パネル」** と **右の「作業エリア」** の2カラムです（画面幅が980px未満になると縦積みに切り替わります）。

```
┌─────────────┬─────────────────────────────────┐
│  操作パネル   │  作業エリア                        │
│ (controls)  │  ┌───────────┬───────────┐        │
│             │  │ Original   │ Reduced    │        │
│ 設定 + ボタン │  ├───────────┴───────────┤        │
│ 進捗 + 状態   │  │ New-color mask          │        │
│             │  ├─────────────────────────┤        │
│             │  │ Exported WebM (書出後表示)│        │
│             │  ├─────────────────────────┤        │
│             │  │ Palette / Metrics / 表    │        │
│             │  └─────────────────────────┘        │
└─────────────┴─────────────────────────────────┘
```

### 4-1. 左カラム：操作パネル（`aside.controls`）

| 表示要素 | 種類 | 役割・表示内容 |
| --- | --- | --- |
| **Palette Reducer** | 見出し (`h1`) | アプリ名。 |
| **Video** | ファイル選択 | 動画ファイルを選ぶ。選ぶとファイル名・形式・サイズが下の状態表示に出る。 |
| **Analysis short side** | 数値入力 | 分析時に画像を縮小する短辺サイズ（既定540）。大きいほど精度↑・処理時間↑。 |
| **Preview short side** | 数値入力 | プレビュー／書き出し時の短辺サイズ（既定360）。大きいほど高画質・重い。 |
| **Max colors** | 数値入力 | 分析に使う候補色の上限（既定800）。増やすほど精度↑・処理時間とメモリ↑。 |
| **Min rep distance** | 数値入力 | 代表色どうしが最低どれだけ離れているべきか（既定12）。近すぎる色の重複を防ぐ。 |
| **Min K / Max K** | 数値入力×2 | 自動で選ぶ代表色数の下限・上限（既定4〜40）。 |
| **Threshold mode** | プルダウン | しきい値の決め方。`Auto`（自動）／`Fixed`（固定）。 |
| **Fixed threshold** | 数値入力 | `Fixed` のときに使うしきい値（既定60）。 |
| **Auto margin** | 数値入力 | `Auto` のときに自動算出値へ上乗せする余裕（既定5）。 |
| **Analyze** | ボタン | 分析を開始。動画を選ぶまで押せない。 |
| **Stop** | ボタン | 分析・プレビュー・書き出し・変換を中断。 |
| **Preview** | ボタン | 3画面の同期プレビューを開始／停止（押すとラベルが Stop に変化）。 |
| **Export WebM** | ボタン | 削減後動画を WebM で書き出し。 |
| **進捗バー** (`progress`) | プログレスバー | 0〜100% で処理の進み具合を表示。 |
| **状態テキスト** (`.status`) | テキスト | 「いま何をしているか」を文章で表示（例: `Extracting first frame`、経過秒つき）。エラーもここに出る。 |
| **Download WebM** | リンク | 書き出し完了後に出るダウンロードリンク（既定は隠れている）。 |

> **ボタンの活性状態**: 起動直後はほぼすべて無効。動画を選ぶと `Analyze` が有効になり、分析が終わると `Preview` と `Export WebM` が有効になります。`Stop` は処理中だけ有効です。この「いつ押せるか」がコードで細かく制御されています。

### 4-2. 右カラム：作業エリア（`section.workspace`）

#### ① 動画3画面（`.video-grid`）
| パネル | 表示内容 |
| --- | --- |
| **Original** | 元動画のフレーム（キャンバス描画）。 |
| **Reduced** | 代表色に置き換えた後の映像。どの代表色にも遠い画素はマゼンタになる。 |
| **New-color mask** | 「新しい色」と判定された画素だけをマゼンタ、それ以外を黒で表示するマスク。どこに想定外の色があるか一目で分かる。 |

3つは `Preview` 中に同期再生されます。Original/Reduced が上段に並び、New-color mask は下段に横幅いっぱいで表示されます。

#### ② Exported WebM（`#outputPanel`、書き出し後に出現）
書き出した WebM を、その場で再生できる動画プレイヤー（`controls` 付き）。書き出すまでは非表示。

#### ③ Palette（`#palette`）
選ばれた代表色を**色見本（スウォッチ）**で一覧表示。各見本に「番号 + HEX値（例: `1. #1A2B3C`）」が付く。

#### ④ Metrics（`#metrics`）— 分析結果の数値サマリ
4列のカードで以下を表示:

| 表示名 | 意味 |
| --- | --- |
| **Colors** | 自動選択された代表色数（K）。 |
| **Threshold** | 採用されたしきい値。 |
| **Unique RGB** | 元の動画（2フレーム）に存在したユニーク色数。 |
| **Coverage** | 候補色が全体の何%をカバーしているか。 |
| **Candidates** | 実際に分析に使った候補色数。 |
| **Min rep dist** | 代表色間の最小距離（実測値）。 |
| **Mode** | しきい値の決め方（auto / fixed）。 |
| **Quantile** | Auto 時の分位点距離（Fixed 時は `-`）。 |

#### ⑤ K candidates（`#candidateTable`）— 代表色数の候補表
代表色数 K ごとに「**K / Relative gap（まとまりの良さ）/ Min rep distance（最小代表色間距離）**」を一覧表示。relative gap の大きい順に並び、**実際に選ばれた K の行はハイライト**されます。「なぜこの色数になったか」を確認するための表です。

## 5. 3D 色空間ビュー（`views/…html`）

`views/interactive_all_unique_threshold_toggle.html` は、本体アプリとは独立した **Plotly 製のインタラクティブ3Dグラフ**です（単体でブラウザに開いて使う参考資料）。

- RGB 立方体の中に、**全ユニーク色**・**代表色**をプロット。
- 代表色のまわりに「しきい値の半径」を球（リング）で表示し、**`no rings` / `threshold 45.37` / `threshold 60`** などのトグルで切替できる。
- 直交投影（ortho）切替や HEX 表示も持つ。

「しきい値を変えると、どの色が代表色の球に入る／はみ出すのか」を立体的に理解するための可視化です。ファイルサイズが大きい（約27MB）のは、色データと Plotly 本体を1ファイルに丸ごと埋め込んでいるためです。

## 6. 制約・注意点（`README.md` より）

- サーバー処理はなし。動画はアップロードされず、ブラウザ内で処理。
- 対応ブラウザは **Chrome / Edge を優先**。
- 書き出しは **MP4 ではなく WebM**。
- 大きい動画ほど処理に時間がかかる。
- 非対応動画の自動変換では、初回に **約31MB の WASM** を読み込む。
- ffmpeg.wasm でもデコードできない特殊な動画は処理不可。
- Python 版と**完全に同じ結果は保証されない**。

### 推奨初期設定
- Analysis short side: `540` / Preview short side: `360` / Max colors: `800` / Min rep distance: `12` / Threshold mode: `Auto`
- 精度を上げたいときは `Max colors` と `Analysis short side` を上げる（その分重くなる）。
