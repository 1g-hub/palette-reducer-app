# GUI リデザイン依頼書 — Browser Palette Reducer

> このドキュメントは、**デザイナーにそのまま渡す**ことを想定した発注書です。
> 目的・ターゲット・現状の課題・必要な GUI 要素・文言・画面遷移・成果物までを一通りまとめています。
> アプリの中身（何をするツールか）は `app-guide.md` を参照してください。

---

## 0. ひとことで言うと

**「動画を選んで、ボタンを順に押すだけで、色数を減らした動画ができる」** と感じられる画面にしたい。
現状は機能は揃っているが、**専門用語と数値入力が多すぎて、初心者（特に日本語話者の非エンジニア）には何をすればいいか分からない**。これを直感的にする。

---

## 1. ターゲットユーザー

- **日本人の初心者**（デザイナー／クリエイター／事務職など、エンジニアではない人）。
- 画像・動画の「色を減らす」というイメージは持っているが、**クラスタリング・しきい値・分位点などの専門用語は知らない**。
- 英語 UI に苦手意識がある。
- スマホよりは **PC（Chrome / Edge）** での利用が中心だが、レスポンシブも欲しい。

### ペルソナ例
> 「動画の色を◯色くらいに統一したい。難しい設定は分からないので、いい感じに自動でやってほしい。仕上がりは見て確認したいし、ダウンロードもしたい。」

---

## 2. 現状の課題（リデザインの理由）

実際の現在の画面（`index.html`）を見たうえでの問題点です。デザインで解決してほしいポイントとして共有します。

1. **専門用語が英語のまま大量に並ぶ** — `Analysis short side` `Min rep distance` `Min K / Max K` `Threshold mode` `Auto margin` `Quantile` など。初心者には意味も、どう変えればいいかも不明。
2. **入力欄が10個以上、いきなり全部見える** — 何を触ればいいか分からず、最初の一歩で止まる。
3. **手順（順番）が画面から読み取れない** — 「①動画を選ぶ→②分析→③確認→④書き出し」という流れが視覚化されていない。ボタンの有効/無効でしか表現されていない。
4. **専門的な出力（Metrics 8項目、K candidates 表）が常に前面に出ている** — 初心者には不要な情報が画面の主役になっている。
5. **状態・進捗が地味** — テキスト1行と細い進捗バーのみ。「いま動いてるのか／終わったのか／失敗したのか」が分かりにくい。
6. **3画面（Original / Reduced / Mask）の意味が伝わらない** — 特に「マゼンタ＝想定外の色」というルールがどこにも説明されていない。
7. **エラー文が技術的** — `Could not load video metadata...` のような英語の技術メッセージがそのまま状態欄に出る。

---

## 3. デザインの基本方針

1. **手順をそのまま画面にする** — 「①読み込む → ②色を減らす → ③確認する → ④保存する」というステップ型 UI（ウィザード/縦フロー）。今どのステップにいるかを常に表示。
2. **専門用語を日本語の平易な言葉に翻訳**し、英語の正式名称は補足（ツールチップ）に逃がす。
3. **「かんたんモード」を初期表示**にし、細かい設定は **「詳細設定」アコーディオンに畳む**。初心者はボタンを押すだけで完結できる。
4. **プリセット（おすすめ設定）**を用意して、数値を1つも触らずに始められるようにする。
5. **結果の見せ方を「絵で理解」優先**に。数値表は詳細モードへ。
6. **状態のフィードバックを強く**（進捗・成功・失敗・所要時間）。
7. **安心感を最初に伝える** — 「動画はアップロードされません。すべてあなたのPCの中で処理されます」を目立つ位置に。

## 4. 専門用語 → やさしい日本語 対応表（コピーガイド）

ラベルはこの日本語を使ってください。英語の正式名称はツールチップ（ⓘ）に入れます。**機能や数値の挙動は一切変えない**（ラベルと見せ方だけ変える）。

| 現状の英語ラベル | 推奨日本語ラベル | ツールチップ（補足説明） |
| --- | --- | --- |
| Video | 動画を選ぶ | 手元の動画ファイルを選びます。アップロードされません。 |
| Analyze | 色を分析する | 動画の色を調べて、代表的な色を選び出します。 |
| Preview | 仕上がりを確認 | 元動画・色を減らした映像・はみ出し色を並べて再生します。 |
| Export WebM | 動画を書き出す | 色を減らした動画を WebM 形式で保存します。 |
| Stop | 中断 | 実行中の処理を止めます。 |
| Download WebM | ダウンロード | 書き出した動画を保存します。 |
| Max colors | 使う色の細かさ（候補数） | 分析に使う色の数。多いほど高精度ですが遅くなります（既定800）。 |
| Analysis short side | 分析の解像度 | 分析時に縮小するサイズ。大きいほど精密ですが遅くなります（既定540）。 |
| Preview short side | プレビュー/書き出しの解像度 | 表示・保存する映像の大きさ（既定360）。 |
| Min rep distance | 似た色をまとめる強さ | 値が大きいほど、近い色どうしを1色にまとめます（既定12）。 |
| Min K / Max K | 代表色の数（最小〜最大） | 自動で選ぶ色数の範囲（既定4〜40）。 |
| Threshold mode | はみ出し色の判定方法 | 「自動」か「手動」。自動がおすすめ。 |
| Fixed threshold | 手動のしきい値 | 手動時に使う基準値（既定60）。 |
| Auto margin | 自動判定の余裕 | 自動時の感度調整。大きいほど“はみ出し色”を少なく判定（既定5）。 |
| New-color mask | はみ出し色マップ | パレットに無い色（＝マゼンタで表示）がどこにあるかを示します。 |
| Reduced | 色を減らした映像 | 代表色だけで作り直した映像。 |
| Original | 元の動画 | 読み込んだそのままの映像。 |
| Palette | 代表色（パレット） | 選ばれた代表色の一覧。 |
| Metrics | 分析の詳細データ | 上級者向けの数値情報。 |
| K candidates | 色数の候補一覧 | なぜこの色数になったかの根拠（上級者向け）。 |
| Threshold / Quantile / Coverage 等 | （詳細モードのみ表示） | 用語は詳細パネル内で個別に補足。 |

**重要な概念の説明文（画面内に常設したい一文）:**
> 🟣 マゼンタ（鮮やかなピンク）は「選んだ代表色のどれにも近くない色」＝**はみ出し色**を表します。

## 6. 必要な GUI 要素一覧（コンポーネント仕様）

デザインしてほしい部品の一覧です。**状態（state）** まで含めて指定してください。既存ロジックに合わせ、機能の取捨選択はせず「見せ方の再設計」に集中します。

### 6-1. グローバル
- **ヘッダー**: アプリ名 + サブタイトル + 🔒プライバシー一文（「アップロードされません」）。
- **ステップインジケータ**: 1〜4の進行表示。状態＝`未着手 / 実行可能 / 進行中 / 完了 / エラー`。現在地を強調。
- **プライバシーバッジ**: 常時見える安心要素。

### 6-2. STEP1 動画選択
- **ドロップゾーン**（ドラッグ＆ドロップ + クリックでファイル選択）。状態＝`空 / ホバー / 読込中 / 選択済み / エラー`。
- **選択ファイル表示**: ファイル名・サイズ・形式・サムネ（最初のフレーム）。
- **差し替えボタン**（別の動画を選ぶ）。

### 6-3. STEP2 設定 & 分析
- **プリセット選択**（ラジオ/セグメント）: `おすすめ` / `高精細（遅い）` / `軽量（速い）`。既定は「おすすめ」。
  - 各プリセットは内部で既存の数値（Max colors / Analysis short side など）にマッピング。値はエンジニア側が設定するので、**デザインはUIのみ**でOK。
- **「色を分析する」主ボタン**（画面で最も目立つ第一アクション）。
- **詳細設定アコーディオン**（既定で閉じる）。中に全数値入力をグルーピングして配置:
  - グループA「精度と速度」: 分析の解像度 / 使う色の細かさ / プレビュー解像度
  - グループB「色のまとめ方」: 似た色をまとめる強さ / 代表色の数（最小〜最大）
  - グループC「はみ出し色の判定」: 判定方法（自動/手動）/ 手動のしきい値 / 自動判定の余裕
  - 各入力に **ⓘツールチップ**、可能なら **スライダー + 数値** の併用、`リセット`ボタン。
  - 「手動」を選んだときだけ手動しきい値を出すなど、**条件表示**を指定。

### 6-4. 進捗・状態フィードバック（全ステップ共通）
- **進捗バー**（％表示 + 大きめ）。
- **状態メッセージ**（日本語・平易）: 例「最初のフレームを読み込み中…」「色をまとめています…（12秒経過）」。
- **長時間時の表示**: 8秒以上応答が無いとき「まだ処理中です…」のような安心メッセージ（既存挙動あり）。
- **中断ボタン**（進行中のみ表示/活性）。
- **トースト/バナー**: 成功・失敗の明確な通知。

### 6-5. STEP3 結果確認
- **プレビュープレーヤー**: 元 / 色を減らした映像 / はみ出し色マップ の3面。再生・停止コントロールは1つで3面同期。各面にラベル + ⓘ。
- **🟣ルールの凡例**（「マゼンタ＝はみ出し色」）。
- **パレット表示**: 代表色スウォッチ。各色 = 色面 + 番号 + HEX、**クリックでHEXコピー**（コピー時トースト）。色数バッジ（例「12色」）。
- **詳細データ（折りたたみ）**: Metrics 8項目（カード）と K candidates 表。上級者向けと明記。

### 6-6. STEP4 書き出し
- **書き出し実行ボタン**（WebM）。
- **完了パネル**: 書き出し動画のプレーヤー + サイズ表示 + **ダウンロードボタン** + 「やり直す」。
- 形式が WebM である旨の小さな注記（MP4ではない）。

### 6-7. 例外・特殊状態
- **空状態（初回）**: 何をすればいいかの一言ガイド。
- **コーデック変換中**: 「この動画は変換が必要です。初回のみ約31MBを読み込みます…」という専用の説明 + 進捗。
- **エラー表示**: 技術メッセージをそのまま出さず、**原因＋次の一手**を日本語で提示。
  - 例: 「この動画を読み込めませんでした。別の形式（MP4）でお試しください。」+ 折りたたみで技術詳細。
- **非対応ブラウザ/環境**: 「Chrome または Edge を推奨します」案内。

## 7. 文言（マイクロコピー）サンプル

そのまま使える日本語コピー案です。トーンは「やさしく・短く・命令形を避ける」。

- 初期: 「まずは動画を選びましょう。」
- 分析前: 「おすすめ設定で色を分析します。設定を変えたい場合は『詳細設定』を開いてください。」
- 分析中: 「色を調べています…（{秒}秒経過）」
- 完了: 「{N}色の代表色を選びました。仕上がりを確認しましょう。」
- マスク凡例: 「🟣 のピンクは、選んだ色に当てはまらない『はみ出し色』です。」
- 書き出し完了: 「書き出しが完了しました（{サイズ}）。ダウンロードできます。」
- エラー: 「うまく読み込めませんでした。MP4 形式の動画でもう一度お試しください。」

## 8. ビジュアル / トーン要件

- **言語**: 日本語主体。英語の専門語はツールチップ補助に限定。
- **トーン**: 親しみやすく、でも信頼できる。装飾しすぎない。
- **配色**: 既存はライトテーマ（背景 `#eef1f4`、アクセント青 `#245f96`、成功緑 `#1f7a4f`）。これをベースに刷新可。
  - **注意**: マゼンタ `#FF00FF` は「はみ出し色」を意味する**機能色**なので、UI のアクセント色として使わないこと（混同を避ける）。
- **タイポ**: 日本語可読性重視（適切な行間・字間）。数値はタブラー数字。
- **アイコン**: ステップ・各アクションに分かりやすいアイコン。
- **状態の色分け**: 進行中（青）/ 完了（緑）/ エラー（赤）/ 中立（グレー）を統一。

## 9. レスポンシブ / アクセシビリティ

- **PC（横長）**: 左に手順ガイド、右に結果。
- **タブレット/狭い画面（〜980px）**: 縦1列に再構成（既存のブレークポイントあり）。
- **スマホ（〜640px）**: 1カラム、ボタンは縦積み・大きめタップ領域。
- **アクセシビリティ**: ボタンに明確なラベル、フォーカスリング、十分なコントラスト比、色だけに依存しない状態表現（アイコン/テキスト併記）、キーボード操作対応。
- 動画3面はデータ量が多いので、**読み込み中スケルトン**を用意。

## 11. 守ってほしい制約（重要）

デザインは自由ですが、**裏側の処理は変えられません**。以下は仕様として固定です。

1. **すべてブラウザ内処理**（サーバーなし・アップロードなし）。これはセールスポイントなので前面に。
2. **入力できる設定値は現状の項目がすべて**（増減はエンジニアと要相談）。ラベルと見せ方の再設計が主目的。
3. **出力は WebM**（MP4 ではない）。UI 上もそう表示。
4. **非対応動画は ffmpeg.wasm で変換**（初回約31MB）→ この待ち時間用の専用 UI が必要。
5. **3つの映像（元 / 削減後 / はみ出し色マップ）と、代表色パレット、進捗、状態テキスト**は機能上必須。隠してよいのは Metrics と K candidates（詳細送り）まで。
6. 対応ブラウザは **Chrome / Edge 優先**。
